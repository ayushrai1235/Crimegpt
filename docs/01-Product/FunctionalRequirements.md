# Functional Requirements

## Overview
The **Functional Requirements** document details the specific technical behaviors, processes, and capabilities the **CrimeGPT** system must possess. It translates the high-level Epics and User Stories into concrete engineering requirements, heavily emphasizing the **Zoho Catalyst-first** architecture.

## 1. Authentication & Authorization (Catalyst Authentication)

### FR-1.1: Login and Session Management
- **Requirement:** The system MUST authenticate users exclusively using **Catalyst Authentication**. 
- **Details:** Passwords must be hashed. Sessions must timeout after 30 minutes of inactivity. Concurrent logins from different IP addresses must be flagged and logged.

### FR-1.2: Role-Based Access Control (RBAC)
- **Requirement:** The system MUST enforce strict RBAC at the API level (via **Catalyst Functions** middleware).
- **Details:** 
  - **Administrator:** Can manage users. Cannot view case evidence.
  - **Investigating Officer:** Can view all FIRs and evidence. Can chat with CrimeGPT.
  - **Commissioner:** Can view macro dashboards. Cannot edit individual case files.

## 2. Conversational AI (CrimeGPT via Catalyst Functions)

### FR-2.1: Query Processing
- **Requirement:** The system MUST process natural language queries, determine the intent (e.g., "Search FIR", "Analyze Network", "Predict Crime"), and route the query to the appropriate tool/database.
- **Details:** This routing logic MUST reside entirely within **Catalyst Functions**.

### FR-2.2: Retrieval-Augmented Generation (RAG)
- **Requirement:** The system MUST ground all LLM responses in verified KSP data.
- **Details:** When answering a factual query about a case, the Catalyst Function must retrieve the relevant document chunks from the Vector DB, pass them as context to the LLM, and force the LLM to cite the source file stored in the **Catalyst File Store**.

### FR-2.3: Chat History Management
- **Requirement:** The system MUST save all conversational context.
- **Details:** Chat sessions must be stored in the **Catalyst Data Store**. Context for active sessions must be temporarily stored in **Catalyst Cache** to reduce database read latency during rapid conversation.

## 3. Data Management (Catalyst Data & File Store)

### FR-3.1: Immutable Audit Logging
- **Requirement:** The system MUST record every significant user action.
- **Details:** Any read of a sensitive FIR, AI query, or file download must append a record to an audit table in the **Catalyst Data Store**. This table MUST NOT allow UPDATE or DELETE operations (enforced at the Catalyst level).

### FR-3.2: Secure Document Storage
- **Requirement:** Raw FIRs, evidence photos, and generated reports MUST be stored securely.
- **Details:** Stored exclusively in the **Catalyst File Store**. The frontend must never have direct URL access to the bucket; access must be mediated via secure, expiring URLs generated by Catalyst Functions.

## 4. Analytics and Background Processing (Catalyst Cron & Cache)

### FR-4.1: Daily Heatmap Generation
- **Requirement:** The system MUST generate updated predictive crime heatmaps daily.
- **Details:** A **Catalyst Cron** job must trigger at 00:00 IST daily. It will execute a Catalyst Function that runs the predictive model, updates the heatmap data, and stores the serialized JSON result in **Catalyst Cache** for instant frontend retrieval.

### FR-4.2: Automated Notifications
- **Requirement:** The system MUST alert specific officers regarding critical events (e.g., suspect released on bail).
- **Details:** Upon a trigger event in the Catalyst Data Store, a Catalyst Function must push a notification to the relevant officer's dashboard via WebSockets or email.

## 5. Search and Indexing (Catalyst Search)

### FR-5.1: Global Text Search
- **Requirement:** The system MUST allow rapid keyword searching across all structured data (FIR metadata, User profiles).
- **Details:** Must utilize **Catalyst Search** indexing to ensure sub-500ms response times for exact string matches and wildcards, independent of the semantic Vector DB search used by the AI.
