# Explainable AI (XAI)

## 1. Overview
CrimeGPT strictly adheres to Explainable AI principles. Every insight, prediction, and summary generated by the platform must be fully transparent, traceable, and evidence-backed to ensure it is court-admissible and reliable for law enforcement.

## 2. Purpose
To eliminate AI hallucination, build trust with investigators, and provide clear reasoning paths for every automated conclusion.

## 3. Functional Requirements
- **FIR Citations**: Every factual claim must be appended with a clickable reference `[FIR-2023-102]`.
- **Confidence Scoring**: All predictions and similarity matches must display a percentage score (0-100%) indicating the model's certainty.
- **Reasoning Path**: The AI must output a step-by-step summary of *how* it reached its conclusion.

## 4. Technical Design

### Evidence Validation Pipeline
The AI Orchestrator enforces XAI through a multi-step validation pipeline:
1. **Retrieval**: Extract raw data from Pinecone (Similar MOs) and Neo4j (Graph connections).
2. **Constrained Generation**: The LLM prompt explicitly forbids generating external facts. It is instructed to output JSON with two primary keys: `"response"` and `"citations"`.
3. **Post-Processing Validation**: A Catalyst function runs a regex check on the `"response"` to ensure every claim maps to an ID present in the `"citations"` array. If an unknown ID is found, the response is flagged as a potential hallucination and blocked.

### Citation Generation Workflow
```mermaid

    LLM[Gemini Generation] --> OutputJSON[{ response: '...', citations: ['FIR-1', 'FIR-2'] }]
    OutputJSON --> Validator[Regex Validator]

    Validator -->|Fail| Fallback[Return Safe Error Message]
```

## 6. Edge Cases
- **Low Confidence Data**: If Pinecone returns matches with a cosine similarity below 0.65, the AI explicitly prefixes its response with "Based on weak circumstantial evidence..."
