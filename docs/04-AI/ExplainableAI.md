# Explainable AI (XAI) and Citations

## Overview
The **Explainable AI (XAI)** document dictates the most critical requirement for the **CrimeGPT** platform: The AI must never be treated as a black box. In law enforcement, an unverified AI claim is worse than useless—it is legally dangerous. 

Every piece of intelligence generated by the system MUST be traceable back to an official document stored securely in the **Zoho Catalyst File Store**.

---

## 1. The Zero Hallucination Mandate
Hallucinations occur when an LLM fabricates plausible-sounding information. The **Catalyst AI Orchestrator** utilizes strict Retrieval-Augmented Generation (RAG) and negative prompt engineering ("Do not guess") to mitigate this. However, mitigation is not proof. XAI provides the proof.

## 2. The Citation Mechanism

### 2.1. Vector DB Metadata Preparation
When an FIR is chunked and stored in the Vector DB during ingestion, its exact source ID is appended as metadata.
```json
{
  "text": "Suspect fled in a white Maruti Swift.",
  "metadata": { "source": "FIR-2023-0105", "page": 2 }
}
```

### 2.2. Prompt Enforcement
When the **Catalyst Function** passes context to the LLM, it injects explicit formatting rules:
> "For every factual claim, you MUST append a citation tag in the format `[source]`. If multiple sources apply, list them as `[source1, source2]`."

### 2.3. Output Example
*CrimeGPT Output:* "The suspect is known to operate in the Koramangala area `[FIR-2021-0045]` and frequently utilizes a white Maruti Swift `[FIR-2023-0105]`."

## 3. UI and Catalyst File Store Integration

The Next.js frontend is programmed to parse the output stream. When it detects the regex pattern `\[(.*?)\]`, it converts it into an interactive UI element.

1. **User Interaction:** The investigator clicks the `[FIR-2023-0105]` badge in the chat window.
2. **Secure Fetch:** The frontend makes an API call to a **Catalyst Function**.
3. **URL Generation:** The Catalyst Function verifies the user's RBAC permissions. If authorized, it generates a secure, short-lived (e.g., 5-minute expiry) download URL using the **Catalyst File Store API**.
4. **Verification:** The original scanned FIR PDF opens in a side-panel within the CrimeGPT interface, allowing the officer to read the exact paragraph the AI used to generate its answer.

## 4. Confidence Scoring (Future Scope)
While current iterations rely on binary citations (it exists in the document or it doesn't), future versions will implement confidence scoring.
- If the AI synthesizes an answer from multiple disparate documents, the **Catalyst Function** will analyze the Cosine Similarity scores of the retrieved chunks.
- If the scores are borderline (e.g., 0.76), the UI will display the answer with a yellow "Low Confidence" warning badge, prompting the officer to manually verify the attached citations before proceeding.
