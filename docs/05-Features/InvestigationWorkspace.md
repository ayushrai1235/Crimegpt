# Feature: Investigation Workspace

## 1. Purpose
The **Investigation Workspace** is a dedicated dashboard tailored for a specific, ongoing case. Instead of querying the entire state database, this workspace isolates all evidence, chat history, and generated graphs pertaining to a single active investigation, reducing cognitive overload for the Investigating Officer (IO).

## 2. Target Users
- Investigating Officers (IOs)
- Supervisory Officers (DSPs) reviewing case progress

## 3. User Journey
1. The officer clicks on an active FIR from their main dashboard.
2. The UI routes to `/cases/[fir_id]`.
3. The Workspace loads, displaying a split-screen view:
   - **Left Panel:** Case metadata, suspect list, attached PDFs, and the specific Criminal Network graph for this case.
   - **Right Panel:** A localized instance of CrimeGPT, specifically tuned to only answer questions about the evidence in this specific workspace.

## 4. Technical Workflow

### 4.1. Localized RAG Context
When CrimeGPT is used within an Investigation Workspace, the **Catalyst Function** alters its behavior.
- **Global Chat:** Searches the entire Vector DB.
- **Workspace Chat:** The Catalyst Function applies a hard metadata filter (`"fir_id": current_case_id`) to the Vector DB query. This ensures the AI only considers documents explicitly linked to this case, ensuring absolute precision and preventing cross-contamination from other cases.

### 4.2. File Management (Catalyst File Store)
- The Workspace allows the officer to upload new evidence (e.g., a newly transcribed interrogation report).
- The Next.js frontend uploads the file directly to the **Catalyst File Store** (via a secure signed URL generated by a Catalyst Function).
- This upload triggers the Event Flow, instantly ingesting the new document into the Vector DB, making it immediately available for the Workspace CrimeGPT to analyze.

## 5. UI/UX Design Considerations
- **Split-Pane Layout:** Investigators need to read the source PDF while simultaneously talking to the AI. The UI must support resizable panes so the PDF viewer and the Chat window can be viewed side-by-side.
- **Evidence Pinning:** Users can "pin" specific AI answers or generated graphs to an "Investigation Board" within the workspace for easy reference later.

## 6. Edge Cases
- **Archived Cases:** If a case is marked as 'Closed' in the **Catalyst Data Store**, the Investigation Workspace becomes Read-Only. The UI hides upload buttons, and the chat interface is disabled to preserve the integrity of the closed case file.
